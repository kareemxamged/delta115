# خطة توزيع المهام - فريق من 5 مطورين (Task Distribution Plan)

## 👥 نظرة عامة
بما أن الفريق يتكون من 5 مطورين نشطين لهذا المشروع، تم تقسيم العمل لضمان التوازي (Parallelism) وتقليل التضارب (Conflicts).

### 🔷 الهيكلية المقترحة للفريق
1. **Frontend Lead & Core** (المطور 1): المسؤول عن البنية الأساسية والمكونات المشتركة.
2. **Student Features Owner** (المطور 2): المسؤول عن تجربة الطالب كاملة.
3. **Teacher Features Owner** (المطور 3): المسؤول عن تجربة المدرس وإنشاء الامتحانات.
4. **Admin & Data Owner** (المطور 4): المسؤول عن لوحة الإدارة والتقارير.
5. **Backend & Integration Specialist** (المطور 5): المسؤول عن Supabase والربط والأمان.

---

## 📅 توزيع المهام التفصيلي

### 👨‍💻 المطور 1: البنية والأساسيات (Frontend Lead)
**التركيز:** التأكد من أن التطبيق يعمل بسلاسة وتوحيد التصميم.
- **المهام:**
  - [ ] إعداد ملفات CSS Variables و Design System (Buttons, Inputs, Cards).
  - [ ] بناء الـ Layouts الرئيسية (Sidebar, Navbar) والمتجاوبة (Responsive).
  - [ ] إدارة نظام التوجيه (React Router) وحماية المسارات (Protected Routes).
  - [x] بناء صفحات تسجيل الدخول والتسجيل (Authentication Pages).
  - [ ] مراجعة الـ Pull Requests لضمان جودة الكود.

### 👨‍💻 المطور 2: وحدة الطالب (Student Module)
**التركيز:** واجهة الطالب وتجربة أداء الامتحان.
- **المهام:**
  - [x] لوحة تحكم الطالب (عرض الامتحانات المتاحة، الامتحانات القادمة).
  - [ ] **محرك الامتحان (Exam Engine)**: أهم جزء (عرض الأسئلة، المؤقت Timer، التنقل بين الأسئلة).
  - [ ] حفظ الإجابات مؤقتاً (Handling Draft Answers).
  - [ ] صفحة عرض النتائج وتفاصيل الإجابات بعد التصحيح.

### 👨‍💻 المطور 3: وحدة المدرس (Teacher Module)
**التركيز:** أدوات المدرس وبناء الامتحانات.
- **المهام:**
  - [ ] لوحة تحكم المدرس (إدارة الامتحانات المنشأة).
  - [ ] **منشئ الامتحانات (Exam Builder)**: واجهة لإضافة الأسئلة (MCQ, True/False, Text).
  - [ ] إدارة بنك الأسئلة (Question Bank).
  - [ ] واجهة التصحيح اليدوي للأسئلة المقالية.
  - [ ] إعدادات الامتحان (وقت البدء، المدة، درجات النجاح).

### 👨‍💻 المطور 4: الإدارة والتقارير (Admin & Analytics)
**التركيز:** إدارة البيانات وعرض الإحصائيات.
- **المهام:**
  - [ ] لوحة تحكم الأدمن (Overview Dashboard).
  - [ ] جداول إدارة المستخدمين (CRUD Operations: Create, Read, Update, Delete).
  - [ ] الرسوم البيانية (Charts) لنتائج الطلاب ونسب النجاح.
  - [ ] تصدير البيانات (Export to Excel/PDF).
  - [ ] نظام الإشعارات (UI Implementation).

### 👨‍💻 المطور 5: قاعدة البيانات والربط (Backend & DevOps)
**التركيز:** Supabase، الأمان، وإدارة البيانات.
- **المهام:**
  - [ ] تصميم وبناء Database Schema والجداول والعلاقات.
  - [ ] كتابة سياسات الأمان (RLS Policies) - من يرى ماذا؟.
  - [ ] كتابة Database Functions & Triggers (مثل التصحيح التلقائي).
  - [ ] إدارة Typescript Types وتحديثها للفريق.
  - [ ] اختبارات الـ API والأداء.

---

## 🔄 سير العمل المقترح (Workflow)
1. **المطور 5** يجهز الجدول في Supabase ويحدث ملف `types.ts`.
2. **المطور 1** يجهز المكونات الأساسية (مثل الزر وحقل الإدخال).
3. **المطورين 2, 3, 4** يبدؤون في بناء صفحاتهم باستخدام المكونات والبيانات الجاهزة.

## ⚠️ نصائح للعمل الجماعي
- **Git Branching**: كل Feature في Branch مستقل (مثلاً `feature/student-exam`).
- **تحديث типаت البيانات**: المطور 5 يجب أن يبلغ الجميع عند تغيير أي شيء في الداتابيس.
- **المكونات المشتركة**: لا تقم ببناء زر خاص بك، استخدم الزر الذي بناه المطور 1.
