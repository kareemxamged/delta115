# Phase 2: Authentication & Roles (المرحلة 2: نظام المصادقة)

## 🎯 الهدف
تأمين التطبيق وتنفيذ نظام تسجيل الدخول المبني على الأدوار (Role-Based Access Control). حيث يتم توجيه كل مستخدم (طالب، مدرس، أدمن) للوحة التحكم الخاصة به مع حماية المسارات الحساسة.

## ✅ المتطلبات
- Phase 1 مكتملة (قاعدة البيانات مهيئة).
- حساب Supabase فعال.

## 📝 الخطوات المنفذة

### 1. طبقة الخدمات (`src/services/auth.ts`)
- تم بناء دوال `signIn` و `signUp` و `signOut` التي تتواصل مع Supabase.
- تم إضافة دالة `getCurrentProfile` التي لا تجلب فقط المستخدم، بل تجلب دوره (`role`) من جدول `profiles`.

### 2. إدارة الحالة (`src/hooks/useAuth.tsx`)
- تم إنشاء `AuthProvider` يغلف التطبيق بالكامل.
- يوفر الـ Context حالة المستخدم الحالية (`user`) لجميع المكونات.
- يستمع الـ Provider للتغيرات اللحظية في حالة المصادقة (مثلاً عند انتهاء الجلسة).

### 3. حماية المسارات (`src/components/PrivateRoute.tsx`)
- مكون ذكي يقوم بفحص:
  1. هل المستخدم مسجل دخول؟ (إذا لا -> توجيه لصفحة Login).
  2. هل المستخدم يملك الدور المطلوب؟ (مثلاً طالب يحاول دخول صفحة مدرس -> توجيه لصفحة Unauthorized).

### 4. واجهة المستخدم (Pages & Layout)
- **Login Page**: نموذج تسجيل دخول متكامل مع معالجة الأخطاء.
- **Register Page**: نموذج تسجيل حساب جديد للطلاب.
- **Layout**: هيكل رئيسي يحتوي على Sidebar يتغير محتواها بناءً على دور المستخدم.

## 📁 الملفات المضافة
| الملف | الوصف |
|-------|-------|
| `src/services/auth.ts` | دوال الاتصال بـ Supabase Auth |
| `src/hooks/useAuth.tsx` | React Context لإدارة جلسة المستخدم |
| `src/components/PrivateRoute.tsx` | حماية المسارات |
| `src/pages/Login.tsx` | صفحة تسجيل الدخول |
| `src/pages/Register.tsx` | صفحة التسجيل |
| `src/App.tsx` | إعداد التوجيه (Routing Configuration) |

## 🧪 كيفية الاختبار

### 1. اختبار تسجيل مستخدم جديد
1. اذهب إلى `/register`.
2. أدخل بيانات جديدة (البريد، كلمة المرور، الاسم).
3. اضغط "Sign Up".
4. يجب أن يتم توجيهك (أو يطلب منك تأكيد البريد إذا كان مفعلاً في Supabase).

### 2. اختبار تسجيل الدخول
1. اذهب إلى `/login`.
2. أدخل بيانات صحيحة.
3. سيقوم النظام بتحويلك للوحة التحكم المناسبة لدورك (Student Dashboard افتراضياً).

### 3. اختبار الحماية
1. سجل دخول كطالب.
2. حاول كتابة الرابط `/admin/dashboard` في المتصفح يدوياً.
3. يجب أن يتم تحويلك إلى `/unauthorized` أو الصفحة الرئيسية.
