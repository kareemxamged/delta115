# Phase 3: Student Dashboard & Design System (المرحلة 3: لوحة الطالب والتصميم)

## 🎯 الهدف
إطلاق الواجهة الخاصة بالطالب، مع تطبيق "نظام التصميم الاحترافي" (Design System) المبني على الزجاج الداكن (Glassmorphism) ليعطي انطباعاً عصرياً ومميزاً.

## ✅ المتطلبات
- نظام مصادقة يعمل (Phase 2).
- متصفح يدعم CSS modern properties.

## 📝 الخطوات المنفذة

### 1. نظام التصميم الشامل (Design System)
- **CSS Variables**: تم تعريف لوحة الألوان (`--bg-app`, `--primary`) في `index.css`.
- **Glassmorphism**: إضافة فئات `glass-card` لعمل تأثيرات الزجاج والضبابية.
- **Dark Mode**: اعتماد الثيم الداكن الافتراضي لراحة العين والمظهر الفخم.
- **Typography**: استخدام خط Inter الحديث.

### 2. مكتبة المكونات (Components)
- **Layout**: تحديث القائمة الجانبية والرأسية لتكون شفافة ومتفاعلة.
- **Card**: مكون قابل لإعادة الاستخدام يعرض العناوين والإحصائيات بتصميم موحد.

### 3. صفحات الطالب (Student Pages)
- **Dashboard**:
  - عرض بطاقات إحصائية (Exams Taken, Average Score).
  - شبكة (Grid) تعرض الامتحانات المتاحة مع زر "Start Exam".
- **Exams List**:
  - صفحة مخصصة لعرض كل الامتحانات.
  - إمكانية الفلترة (Pending, Completed).
  - شريط بحث (Search Bar).
- **Profile** (جديد):
  - صفحة شخصية متكاملة لإدارة البيانات والإعدادات.
  - تصميم احترافي (Modern UI) مع إحصائيات بصرية.
- **Calendar & Schedule** (جديد):
  - عرض الامتحانات في تقويم شهري/أسبوعي تفاعلي.
- **Courses** (جديد):
  - مكتبة المساقات المشترك بها.
  - صفحة تفاصيل لكل مساق (Exams, Materials, Grades).
- **Exam Engine** (جديد):
  - واجهة تقديم الامتحان (Full Screen).
  - مؤقت تنازلي (Timer) وحفظ تلقائي للإجابات.

## 🧪 كيفية الاختبار

### 1. المظهر العام (UI Check)
1. سجل دخول كطالب.
2. لاحظ الخلفية الداكنة وتأثير الزجاج في القائمة الجانبية.
3. حرك الماوس فوق البطاقات ولاحظ تأثير الـ Hover (ارتفاع البطاقة).

### 2. لوحة التحكم
1. تأكد من ظهور 3 بطاقات إحصائية في الأعلى.
2. تأكد من ظهور قائمة "Available Exams" في الأسفل.

### 3. قائمة الامتحانات
1. انتقل إلى "My Exams" من القائمة الجانبية.
2. جرب النقر على زر "Completed" لتصفية النتائج.
3. اكتب شيئاً في شريط البحث (شكلي حالياً).
